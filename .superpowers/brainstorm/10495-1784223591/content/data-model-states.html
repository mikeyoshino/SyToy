Section 2 — Data Model & State Transitions

ออกแบบให้ business rule อยู่ใน aggregate และเก็บ snapshot ทุกค่าที่ห้ามเปลี่ยนตามข้อมูลปัจจุบัน

Product Aggregate

  • DisplayName, EnglishName, Slug
  • Description, Category, SaleType
  • Brand, Universe, Characters[]
  • Images[] + ordered primary
  • Draft / Published / Archived
  • InStockOffer หรือ PreOrderOffer เท่านั้น

PreOrderOffer

  • FullPrice + DepositAmount
  • BalanceAmount = Full − Deposit
  • CloseDate @ 23:59 Bangkok
  • ETA month/year
  • TotalCapacity
  • MaxPerCustomer
  • BalancePaymentDays = 7

Inventory Aggregate

  • InventoryItem per Product
  • OnHand / active Reserved
  • StockMovement audit
  • Reservation + expiry
  • Optimistic concurrency
  • Available = OnHand − Reserved

CheckoutAttempt

  • Customer/anonymous owner
  • Price and item snapshot
  • Address snapshot
  • Reservation references
  • Stripe Checkout Session
  • ExpiresAt + idempotency key

Order Aggregate

  • OrderNumber + SaleType
  • Customer + address snapshot
  • OrderItem snapshots
  • PaymentStatus
  • FulfillmentStatus
  • Pre-order policy snapshot
  • Audit trail

Operational Records

  • Payment: Full / Deposit / Balance
  • BalancePaymentRequest
  • Shipment + carrier + tracking
  • EmailDelivery
  • LineDelivery
  • Refund + provider reference
Uniqueness: Product EnglishName and Slug are case-insensitive unique; Brand/Universe names and slugs are unique; Character name is unique within its Universe. Stripe event, Checkout Session and Payment reference each have unique constraints for idempotency.

In-stock Order

CheckoutAttempt + ReservationPaid + ReadyToShipPaid + ShippedCancelled + Refunded (Admin, before shipped)

Pre-order Order

DepositPaid + AwaitingPreOrderArrivalDepositPaid + AwaitingBalancePaymentPaid + ReadyToShipPaid + ShippedCancelled + DepositForfeited (customer/overdue)Cancelled + Refunded (Admin/supplier)
Deletion rule: Product, Brand and Universe ที่ถูกอ้างอิงแล้วใช้ Archive แทน hard delete; Order, Payment, StockMovement, Shipment และ audit records ห้ามลบจากหน้า Admin
